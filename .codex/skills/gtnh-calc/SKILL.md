# GTNH 产线配平计算

## 电压等级

| 等级 | 电压 | 等级 | 电压 | 等级 | 电压 |
|------|------|------|------|------|------|
| LV | 32 | MV | 128 | HV | 512 |
| EV | 2048 | IV | 8192 | LuV | 32768 |
| ZPM | 131072 | UV | 524288 | UHV | 2097152 |
| UEV | 8388608 | UIV | 33554432 | UXV | 134217728 |

UXV 以上按 ×4 递进至 MAX(2147483647)。

## 超频规则

OC 档数 = tier_index(供电电压) - tier_index(配方原始电压)

同等级供电 = 0 档，无 OC。

**有损 OC（默认，所有单方块机器）：**
- EU_final = EU_base × 4^n
- time_final = time_base ÷ 2^n
- n = OC 档数，可叠多档

**无损 OC（部分多方块机器支持）：**
- EU_final = EU_base × 4^n
- time_final = time_base ÷ 4^n
- n = OC 档数，可叠多档

## 产率计算

操作次数/秒 = 1 ÷ time_final (单台)
产出速率 = (产物数量 ÷ 每操作) × 操作次数/秒 × 台数
消耗速率 = (原料数量 ÷ 每操作) × 操作次数/秒 × 台数

## 全局配平标准

忽略物品传输时间，默认产物即时到达下一步。对每种物料汇总：

总产出速率 = Σ(所有生产该物料的步骤产出速率)
总消耗速率 = Σ(所有消耗该物料的步骤消耗速率)

配平通过条件：对每一种物料，总产出速率 ≥ 总消耗速率。

非全产线消耗的副产物不纳入配平（询问用户确认）。

## 交互流程（必须遵守）

### 第一步：建立物料流转图

从原料开始确认产线完整路径结构。记录每种物料的生产者列表和消费者列表，构建全局物料供需关系网。

### 第二步：逐台机器确认（每台按以下顺序）

1. 机器类型：单方块还是多方块？单方块 → 默认有损 OC，跳过步骤 2；多方块 → 进入步骤 2
2. OC 模式（仅多方块）：有损还是无损？
3. 配方数据（双重交叉验证）：先查 Java 源码（参考 references/recipe-sources.md）获取配方，再向用户确认 NEI 中显示的原料输入、产物输出、EU/t、耗时
4. 供应电压电流：电压等级和电流
5. 机器数量：这一步用几台机器并行？

### 第三步：副产物处理

- 默认以主要产物为配平基准
- 若副产物在后续步骤中被消耗，询问用户是否纳入物料流转
- 若副产物不参与后续合成，不计入全局配平

### 第四步：确定主要目标

询问用户以产线哪一步作为主要标准。若指定某一步，则该步的产出速率为主要考虑目标；若未指定，以产线最终产物为主要目标。

### 第五步：全局配平计算

全部确认后调用 scripts/throughput_calc.py。脚本汇总每种物料的全产线产出速率和消耗速率，对比供需。

### 第六步：微调（按以下规则逐轮执行）

1. 检查效率：计算当前配置是否满足全局需求
2. 速率不足时：调整供给电压或机器数量，重新计算，直至速率满足需求
3. 速率明显超出时：若除原料处理步骤外，其他步骤负载均在 50% 以下，视为速率超出。询问用户是否降低配置。若肯定，重新计算直至满足需求
4. 以主要目标为锚：微调时以第四步确定的主要目标为核心
5. 默认目标：若用户未指定，以产线最终产物为主要目标
6. 负载上限：配平结果中每步负载 ≤ 100%

## 计算脚本

scripts/throughput_calc.py — 确定性计算器。纯数学公式实现，同等输入永远同等输出。

输入 JSON 格式见脚本文件 docstring。输出：每步 OC 档数/耗时/速率/负载、全局物料供需表、瓶颈报告、配平结论。
